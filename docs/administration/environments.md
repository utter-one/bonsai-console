# Environments

**Environments** represent deployment targets for your projects — for example, development, staging, and production. They allow you to manage different configurations for different deployment contexts.

## Creating an Environment

Go to **Administration > Environments** and click **Create Environment**.

### Fields

- **Name** — A clear label (e.g., "Development", "Staging", "Production").
- **Description** — Optional notes about what this environment is for.

## Use Cases

- **Development** — For building and testing new conversation flows without affecting live users.
- **Staging** — A pre-production environment for final testing before deployment.
- **Production** — The live environment serving real users.

## Tips

- **Keep environment names consistent** — Use a standard naming convention across your team.
- **Test in staging first** — Always verify changes in a non-production environment before pushing to production.
